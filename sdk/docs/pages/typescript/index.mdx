# Haneul Typescript SDK Quick Start

The Haneul Typescript SDK is a modular library of tools for interacting with the Haneul blockchain. Use it
to send queries to RPC nodes, build and sign transactions, and interact with a Haneul or local network.

## Installation

```sh npm2yarn
npm i @haneullabs/haneul.js
```

## Network locations

The following table lists the locations for Haneul networks.

| Network | Full node                             | faucet                                |
| ------- | ------------------------------------- | ------------------------------------- |
| local   | `http://127.0.0.1:9000` (default)     | `http://127.0.0.1:9123/gas` (default) |
| Devnet  | `https://fullnode.devnet.haneul.io:443`  | `https://faucet.devnet.haneul.io/gas`    |
| Testnet | `https://fullnode.testnet.haneul.io:443` | `https://faucet.testnet.haneul.io/gas`   |
| Mainnet | `https://fullnode.mainnet.haneul.io:443` | `null`                                |

## Migrate to version 0.38.0

The Haneul TypeScript SDK was refactored beginning with version 0.38.0. If you are updating from an
earlier version of the SDK, there are some changes you should consider when updating your code.

### Module structure

The Haneul TypeScript SDK is now divided into modular components. Before version 0.38.0, you imported
the complete SDK module. Now, you upload the individual packages of the SDK module instead. See the
[Architecture section](#architecture) for the list of packages.

### Deprecated classes

The Haneul TypeScript SDK deprecates the following classes with version 0.38.0:

- `JsonRpcProvider` - The `JsonRpcProvider` class is deprecated in favor of the `haneulClient` class
  when creating a client for a Haneul network. See [Network Interactions with HaneulClient](haneul-client)
  for more information.
- `SignerWithProver` and `RawSigner` - Key pairs now directly support signing transactions and
  messages without the need of a `Signer` class. See the [Key pairs](cryptography/keypairs) topic
  for more information.
- `signAndExecuteTransactionBlock` - This method was not deprecated, but is now part of `HaneulClient`.
- `Connection` classes - The `Connection` classes (`Connection`, `devnetConnection`, and so on) have
  been deprecated in favor of using `haneulClient` for establishing the connection. See
  [Network Interactions with HaneulClient](haneul-client) for more information.
- The `superstruct` type definitions for `JsonRPCProvider` types are replaced with generated types
  exported from `@haneullabs/haneul.js/client`. The new type definitions are pure TypeScript types that you
  can't use for runtime validation.
- A more stable JSON-RPC API has reduced the need for many of the SDK "getter" methods, which are
  now deprecated.

### Signing transaction blocks

Signing and sending transaction blocks changes slightly with the deprecation of the `Signer`
pattern. For an example of transaction signing, see the
[Haneul Programmable Transaction Blocks Basics](basics) topic.

### Faucet requests

The ability to request HANEUL from a faucet is not part of `HaneulClient` as it was with
`JsonRpcProvider`. Instead, you must use the `requestHaneulFromFaucetV0` method from
`@haneullabs/haneul.js/faucet`. The `@haneullabs/haneul.js/faucet` import also provides a `getFaucetHost` method
to retrieve the faucet URL for `localnet`, `testnet`, or `devnet` networks.

```ts
import { getFaucetHost, requestHaneulFromFaucetV0 } from '@haneullabs/haneul.js/faucet';

await requestHaneulFromFaucetV0({
	host: getFaucetHost('devnet'),
	recipient: '<HANEUL_ADDRESS>',
});
```

## Module packages

The SDK contains a set of modular packages that you can use independently or together. Import just
what you need to keep your code light and compact.

- [`@haneullabs/haneul.js/client`](/typescript/haneul-client) - A client for interacting with Haneul RPC nodes.
- [`@haneullabs/haneul.js/bcs`](/typescript/bcs) - A BCS builder with pre-defined types for Haneul.
- [`@haneullabs/haneul.js/transaction`](/typescript/transaction-building/basics) - Utilities for building
  and interacting with transactions.
- [`@haneullabs/haneul.js/keypairs/*`](/typescript/cryptography/keypairs) - Modular exports for specific
  KeyPair implementations.
- [`@haneullabs/haneul.js/verify`](/typescript/cryptography/keypairs#verifying-signatures-without-a-key-pair) -
  Methods for verifying transactions and messages.
- [`@haneullabs/haneul.js/cryptography`](/typescript/cryptography/keypairs) - Shared types and classes for
  cryptography.
- [`@haneullabs/haneul.js/multisig`](/typescript/cryptography/multisig) - Utilities for working with
  multisig signatures.
- [`@haneullabs/haneul.js/utils`](/typescript/utils) - Utilities for formatting and parsing various Haneul
  types.
- [`@haneullabs/haneul.js/faucet`](/typescript/faucet) - Methods for requesting HANEUL from a faucet.
- [`@haneullabs/haneul.js/zklogin`](/typescript/zklogin) - Utilities for working with zkLogin. (Note
  @haneullabs/zklogin package also contains zklogin related functionality)
