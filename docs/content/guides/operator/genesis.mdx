Genesis is the initial state of the Haneul blockchain. To launch a network, the initial committee of validators collaborate by providing their validator information (public keys, network addresses, and so on) to a shared workspace. After all of the initial validators have contributed their information, Haneul generates the initial, unsigned genesis checkpoint (checkpoint with sequence number 0) and each validator provides their signature. Haneul aggregates these signatures to form a certificate on the genesis checkpoint. Haneul bundles this checkpoint, as well as the initial objects, together into a single genesis.blob file that is used to initialize the state when running the `haneul-node` binary for both validators and Full nodes.

## Genesis blob locations

The `genesis.blob` files for each network are in the [haneul-genesis](https://github.com/GeunhwaJeong/haneul-genesis) repository.

See [Haneul Full Node](./haneul-full-node.mdx) for how to get the genesis.blob file for each network.
