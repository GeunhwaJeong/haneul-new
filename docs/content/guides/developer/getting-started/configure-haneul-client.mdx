The Haneul client configuration specifies which network to connect to and which address to send transactions. 

<Tabs className="tabsHeadingCentered--small">
<TabItem value="prereq" label="Prerequisites">

First, confirm that Haneul has been installed successfully:

```
$ haneul --version
```

If this command returns `haneul not found`, then Haneul is not installed and you must [follow the installation instructions](/guides/developer/getting-started/haneul-install.mdx).

</TabItem>
</Tabs>

## `haneul client` 

Run the Haneul CLI with the command:

```
$ haneul client
```

:::info 

If the Haneul client is already configured, running this command returns the `haneul client --help` output rather than interactive prompts. If this happens, skip to [learning about the `client.yaml`](#client-yaml) file.

:::

The first prompt confirms if you want to create the `client.yaml` file, select `Y`:

```
Config file ["<PATH-TO-FILE>/client.yaml"] doesn't exist, do you want to connect to a Haneul full node server [y/N]?
```

The second prompt configures the network server URL to connect to:

```
Haneul full node server URL (Defaults to Haneul Testnet if not specified) :
```

If you specify nothing, the client uses Testnet by default. Using Testnet is recommended for developers new to Haneul that do not have a project already developed. 

The options are:

- `https://fullnode.testnet.haneul.io:443`: The RPC URL for Testnet.

- `https://fullnode.devnet.haneul.io:443`: The RPC URL for Devnet. Devnet is not recommended for new Haneul developers, as it is better suited for more advanced use cases and the network wipes data weekly as part of its update process.

- `https://fullnode.mainnet.haneul.io:443`: The RPC URL for Mainnet network. Publishing applications to Mainnet costs real HANEUL tokens. 

- `http://0.0.0.0:9000`: The RPC URL for a Localnet network, if configured. Learn how to [setup a local network](/guides/developer/haneul-101/local-network.mdx).

The final prompt is to select an encryption scheme. You must select a key scheme by pressing <kbd>0</kbd>, <kbd>1</kbd>, or <kbd>2</kbd>. After you select a scheme, the client generates a new Haneul address.

```
Select key scheme to generate key pair (0 for ed25519, 1 for secp256k1, 2 for secp256r1):
```

:::caution 

Store recovery phrases securely and do not share them with anyone, as they provide access to any objects and tokens that an address owns.

It will not be visible again once the CLI history disappears.

[Learn more](/guides/developer/getting-started/get-address.mdx) about Haneul addresses, key generation, and recovery phrases.

:::

## `client.yaml`

Your Haneul client is now configured. By default, Haneul stores this information in either the `~/.haneul/haneul_config/client.yaml` file (macOS/Linux) or `%USERPROFILE%\.haneul\haneul_config\client.yaml` file (Windows). You can store a `client.yaml` file in a different location, if preferred, and specify it's location with the `--client.config` flag. 

Haneul stores the key for the Haneul address in a separate file, `~/.haneul/haneul_config/haneul.keystore` (macOS/Linux) or `%USERPROFILE/.haneul/haneul_config/haneul.keystore` (Windows). Learn more about Haneul addresses in [Create a Haneul Address](/guides/developer/getting-started/get-address.mdx).

:::caution 

If a previous Haneul installation stored a `client.yaml` file locally, you will receive the `haneul client --help` output in the console. You can delete the existing `~/.haneul/haneul_config/client.yaml` file if you'd like to start fresh, or you can continue using the existing configuration. 

:::

The `client.yaml` file contains the following configuration information:

```yaml
keystore:
  File: /Users/your_user/.haneul/haneul_config/haneul.keystore
external_keys: ~
envs:
  - alias: testnet
    rpc: "https://fullnode.testnet.haneul.io:443"
    ws: ~
    basic_auth: ~
active_env: testnet
active_address: "0x7df01d3935b7ab3cd1da6828eed763ee2cc556670528a47baa6af8c251866e48"
```
