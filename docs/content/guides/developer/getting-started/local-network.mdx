---
title: Connect to a Local Network
---

Use a Haneul local network to test your dApps against the latest changes to Haneul, and to prepare for the next Haneul release to the Devnet or Testnet network. To set up a local network, Haneul provides the `haneul-test-validator` binary. The `haneul-test-validator` starts a local network that includes a Haneul Full node, a Haneul validator, and a Haneul faucet. You can use the included faucet to get test HANEUL to use on the local network.

If you haven't already, you need to [install Haneul](./haneul-install.mdx) on your system.

## Start the local network

To start the local network, run the following command from your `haneul` root folder.

```bash
RUST_LOG="off,haneul_node=info" cargo run --bin haneul-test-validator
```

The command starts the `haneul-test-validator`. The `RUST_LOG`=`off,haneul_node=info` turns off logging for all components except
haneul-node. If you want to see more detailed logs, you can remove `RUST_LOG` from the command.

**Important:** Each time you start the `haneul-test-validator`, the network starts as a new network with no previous data. The local network is not persistent.

To customize your local Haneul network, such as changing the port used, include additional parameters in the command to start `haneul-test-validator`:

```
OPTIONS:
        --epoch-duration-ms <EPOCH_DURATION_MS>
            The duration for epochs (defaults to one minute) [default: 60000]

        --faucet-port <FAUCET_PORT>
            Port to start the Haneul faucet on [default: 9123]

        --fullnode-rpc-port <FULLNODE_RPC_PORT>
            Port to start the Fullnode RPC server on [default: 9000]
```

Use `haneul-test-validator --help` to see these options in your console.

### Access your local Full node

Use the following command to retrieve the total transaction count from your local network:

```bash
curl --location --request POST 'http://127.0.0.1:9000' \
--header 'Content-Type: application/json' \
--data-raw '{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "haneul_getTotalTransactionBlocks",
  "params": []
}'
```

If successful, the response resembles the following:

```json
{
	"jsonrpc": "2.0",
	"result": 168,
	"id": 1
}
```

## Connect the Haneul Client CLI to your local network

You can use the Haneul Client CLI with any Haneul network. By default it connects to Haneul Devnet. To connect to your local network, create a new environment alias named `local` that sets the RPC URL the client uses to your local network.

```shell
haneul client new-env --alias local --rpc http://127.0.0.1:9000
```

Next, use the following command to set the active environment to the new `local` environment you created.

```
haneul client switch --env local
```

The command returns:

`Active environment switched to [local]`

You can check the current active environment with the following command:

```bash
haneul client active-env
```

The command returns:

`local`

## Show the current active address

The Haneul Client CLI uses the active address for command if you don't specify one. Use the following command to show the active address on your local network.

```
haneul client active-address
```

The command returns an address:

`0xbc33e6e4818f9f2ef77d020b35c24be738213e64d9e58839ee7b4222029610de`

Use the active address to get test HANEUL to use on your local network. Use the `haneul client addresses` command to see all of the addresses on your local network.

:::info

The address returned when you run the command is unique and does not match the one used in this example.

:::

## Use the local faucet {#use-the-local-faucet}

Transactions on your local network require HANEUL coins to pay for gas fees just like other networks. You can use the currently active address with the faucet.

Haneul CLI provides the `haneul client faucet` command to get coins from the faucet. In the most basic case, run `haneul client faucet` and wait up to 60 seconds for the coins to reach your address. Use `haneul client gas` to check for the new coins.

:::info

The `faucet` command uses the active address and the active network environment by default. If you need to pass in a different address or faucet server URL, check the `help` menu. If you're using a different network than a local network or the public ones (fullnode.network.haneul.io), you will have to pass the URL to the faucet server.

:::

## Generate example data

Use the TypeScript SDK to add example data to your network. Run the following command from the `haneul` root folder:

```bash
pnpm --filter @haneullabs/haneul test:e2e
```

For additional information about example data for testing, see [https://github.com/GeunhwaJeong/haneul/tree/main/sdk/typescript#testing](https://github.com/GeunhwaJeong/haneul/tree/main/sdk/typescript#testing).

## Troubleshooting

If you do not use [Node.js 18](https://nodejs.org/de/blog/announcements/v18-release-announce), you might see the following message:

`Retrying requesting from faucet: Retry failed: fetch is not defined`

To resolve this, switch or update to Node.js 18 and then try again.

## Test with the Haneul TypeScript SDK

The published version of the Haneul TypeScript SDK might be an earlier version than the version of Haneul you installed for your local network. To make sure you're using the latest version of the SDK, use the `experimental`-tagged version (for example, `0.0.0-experimental-20230317184920`) in the [Current Tags](https://www.npmjs.com/package/@haneullabs/haneul/v/0.0.0-experimental-20230127130009?activeTab=versions) section of the Haneul NPM registry.
