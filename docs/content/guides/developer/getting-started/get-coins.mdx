To perform transactions on a Haneul network, you need HANEUL tokens to pay for gas and storage. 

For applications on Mainnet, you must buy real HANEUL on exchanges like Coinbase, OKX, or Robinhood. Its value is determined by market factors such as supply and demand, network utility, and perceived value.

Because Mainnet tokens cost real money, development can be expensive. That's why Haneul provides Devnet and Testnet networks, which mirror Mainnet features and enable you to build and test safely for an affordable cost.

:::important

HANEUL tokens on Devnet and Testnet are free and hold no monetary value.

:::

You can get free HANEUL tokens from the Haneul faucet, which lets you deploy and interact with smart contracts and objects on Devnet, Testnet, or a local network. There is no faucet for Mainnet.

## Request HANEUL tokens from faucet {#online-faucet}

Visit the online faucet to request HANEUL tokens: https://faucet.haneul.io/

Then, follow these steps: 

1. Connect your wallet or paste your wallet address in the address field. Get your wallet address in the CLI with the command `haneul client active-address`. 
1. Use the network dropdown to select the correct network: Testnet, Devnet, or Localnet. It is recommended to use the Testnet for most use cases before deploying to Mainnet for production. [Learn more](/guides/developer/getting-started/configure-haneul-client.mdx) about the available networks and their differences. 
1. Click the **Request HANEUL** button. 

To request more HANEUL, refresh your browser and click the **Request HANEUL** button again. The requests are rate limited, however, so too many requests results in a waiting period before you are able to request more tokens.

### Community faucets 

You can also use the following community-provided faucets to obtain HANEUL tokens:

- http://faucet.n1stake.com/

- http://faucet.haneullearn.io/

These faucets have their own separate limits, such as one request per day or one request every few hours.

### Verify token balance 

To check your balance of HANEUL tokens and confirm you received some from the faucet, use the command:

```
$ haneul client balance 
```

Alternatively, use a Haneul Explorer such as [HaneulVision](https://haneulvision.xyz/) or [HaneulScan](https://haneulscan.xyz/mainnet/accounts) to check the HANEUL token balance of an address.

You should have a balance of HANEUL tokens:

```
╭────────────────────────────────────────────╮
│ Balance of coins owned by this address     │
├────────────────────────────────────────────┤
│ ╭────────────────────────────────────────╮ │
│ │ coin       balance (raw)  balance      │ │
│ ├────────────────────────────────────────┤ │
│ │ Haneul        56804696124     0.50 HANEUL    │ │
│ ╰────────────────────────────────────────╯ │
╰────────────────────────────────────────────╯
```

If you do not have a balance, repeat the [request HANEUL tokens from faucet](#online-faucet) steps, or use an [alternative faucet method](#alternative-faucets).

### Return unused Testnet HANEUL

The faucet drains from a finite pool of HANEUL. If the pool empties, it disrupts faucet service for the rest of the community. To help ensure this doesn't happen, you can use the online faucet to return your unused HANEUL to the Testnet pool. You cannot return HANEUL to the Devnet or Localnet pools.

There are two ways to return unused Testnet HANEUL tokens:

- Connect your wallet to the online faucet, and click the **Return tokens to faucet** button. Approve the transaction using your wallet and your HANEUL tokens are returned to the pool.

- If you prefer not to connect your wallet, send the tokens to the address `0x7a9d19d4c210663926eb549da59a54e25777fef63161bfccda08277b58b4212e` via a [separate transaction](/guides/developer/transactions/sign-and-send-txn.mdx).

## Alternative methods for getting HANEUL tokens {#alternative-faucets}

If you need an alternative method to get HANEUL tokens for deployment on Testnet or Devnet, you have several options.

:::tip

Every method to obtain HANEUL tokens for Testnet and Devnet is rate limited. If you need a large amount of HANEUL tokens, use a local network.

:::

<Tabs>

<TabItem value="discord" label="Discord">

1. Join [Discord](https://discord.gg/haneul).
   
   If you try to join the Haneul Discord channel using a newly created Discord account, you might need to wait a few days for validation.
1. Request HANEUL tokens in the Haneul [#devnet-faucet](https://discord.com/channels/916379725201563759/971488439931392130) or [#testnet-faucet](https://discord.com/channels/916379725201563759/1037811694564560966) Discord channels. Send the following message to the channel, replacing `<YOUR HANEUL ADDRESS>` with your client address:
   ```sh
   !faucet <YOUR HANEUL ADDRESS>
   ```

</TabItem>
<TabItem value="curl" label="cURL">

Use the following cURL commands to set console variables and request tokens directly from the faucet server. Set `NETWORK` to either `testnet` or `devnet`.

```sh
$ ADDRESS=$(haneul client active-address)
$ NETWORK=testnet
$ curl --location --request POST "https://faucet.${NETWORK}.haneul.io/v2/gas" \
--header "Content-Type: application/json" \
--data-raw "{
    \"FixedAmountRequest\": {
        \"recipient\": \"${ADDRESS}\"
    }
}"
```

If you're working with a local network, replace `"https://faucet.${NETWORK}.haneul.io/v2/gas"` with the appropriate value based on which package runs your network:

- `haneul-faucet`: `http://127.0.0.1:5003/gas`
- `haneul`: `http://127.0.0.1:9123/gas`

</TabItem>
<TabItem value="typescript" label="TypeScript SDK">

You can access the faucet using the [Haneul TypeScript SDK](https://sdk.haneul-labs.com/typescript).

```typescript
import { getFaucetHost, requestHaneulFromFaucetV2 } from '@haneullabs/haneul/faucet';

// get tokens from the Devnet faucet server
await requestHaneulFromFaucetV2({
	// connect to Devnet
	host: getFaucetHost('devnet'),
	recipient: '<YOUR HANEUL ADDRESS>',
});
```
</TabItem>
</Tabs>
