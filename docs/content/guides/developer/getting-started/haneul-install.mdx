# Install Haneul

Haneul is a scalable and performant layer-1 blockchain that is home to a complete stack of native primitives ideal for building decentralized applications. Such primitives, such as those for [encryption](https://seal-docs.wal.app/UsingSeal/), [data storage](https://docs.wal.app/usage/setup.html), verification, and access control, provide developers with every piece of the application stack without needing to use layer-2 chains or off-chain solutions. 

In contrast to other chains, Haneul uses an [object-centric model](/concepts/object-model.mdx), where every item on the network is an object. [Transactions](/concepts/transactions.mdx) use objects as input, which mutate an existing object or create new objects. Each object has a unique on-chain ID.

To create objects, submit transactions, and start building an application on Haneul, first you must install Haneul. This installation includes the [Haneul CLI](/references/cli.mdx), a tool that creates and manages address balances, builds and publishes smart contracts, and queries information from the network.

## Quick install 

The most effective way to install the Haneul CLI is to use [`haneulup`](https://github.com/GeunhwaJeong/haneulup), as it allows you to easily install and switch between different versions of not only the Haneul CLI but also other Haneul stack components like [`walrus`](https://docs.wal.app/usage/setup.html) and [`mvr`](https://github.com/GeunhwaJeong/mvr). 

Alternative quick install instructions for [Homebrew](https://brew.sh/) or [Chocolately](https://chocolatey.org/) do not support installing other Haneul stack components. Other components will need to be installed through their individual binaries if you'd like to use them in the future. 

<ImportContent source="quick-install.mdx" mode="snippet" />

:::info

The quick install is suitable for most use cases. For those wanting more control over the installation process, you can [install from source](/guides/developer/getting-started/install-source.mdx) or [install binaries](/guides/developer/getting-started/install-binaries.mdx). 

:::

If Haneul is already installed from a previous development environment, be sure to upgrade to the [latest version](/guides/developer/getting-started/install-binaries.mdx).

### Confirm the installation {#confirm-the-installation}

To confirm that Haneul installed correctly:

1. Open a terminal or console
2. Type `haneul --version` and press Enter

If you receive a "command not found" error, verify the Haneul binaries directory is in your `PATH` environment variable.

:::info Looking for a project to clone?

Before you can create and publish a smart contract, you must [configure a Haneul client](/guides/developer/getting-started/configure-haneul-client.mdx) and [obtain HANEUL tokens](/guides/developer/getting-started/get-coins.mdx).

Then, you can [create a "Hello, World!" example](/guides/developer/getting-started/hello-world.mdx) or [clone another example](/guides/developer/app-examples.mdx) to start building.

:::

### `haneulup` installation details 

`haneulup` copies the binaries to `$HOME/.local/bi`n by default on macOS and Linux systems by default, unless the `HANEULUP_DEFAULT_BIN_DIR` environment variable has been set. macOS and Linux installations also include environment variables for `XDG_DATA_HOME`, `XDG_CACHE_HOME`, and `XDG_CONFIG_HOME`.

On Windows, `haneulup` copies the binaries to `LOCALAPPDATA\bin`, and will use `LOCALAPPDATA` or `USERPROFILE\AppData\Local` for storing data and `TEMP` or `USERPROFILE\AppData\Local\Temp` for caching.

### Default configuration file

Regardless of whether you used `haneulup`, Homebrew, or Chocolately, Haneul will store a primary configuration in the `~/.haneul/haneul_config/client.yaml` file. This file defines settings and preferences for your environment, such as:

- Network environment details for Mainnet, Testnet, Devnet, and Localnet networks.

- Active environment, which specifies the network the CLI commands will target.

- Active address, which specifies the Haneul address the CLI will use for transactions and queries.

- Keystore location, which specifies where Haneul stores your address' private keys.
