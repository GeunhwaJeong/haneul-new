Offline signing is the process of signing transactions using a device that is not connected to a Haneul network or using a wallet implemented in a different programming language that doesn't rely on Haneul Keystore. To implement offline signing:

1. Serialize the data for signing.

1. Place the serialized data in a location, such as the wallet of your choice, or tools in other programming languages, and produce a signature with the corresponding public key. 

1. Sign the data. 

1. Execute the signed transaction.

## Serialize data

You must serialize transaction data following [Binary Canonical Serialization](https://crates.io/crates/bcs) (BCS). It is supported in other languages.

The following example demonstrates how to serialize data for a transfer using the [Haneul CLI](/references/cli.mdx). This returns serialized transaction data in Base64. Submit the raw transaction to execute as `tx_bytes`.

<ImportContent source="info-gas-budget.mdx" mode="snippet" />

```sh
$ haneul client transfer-haneul --to <HANEUL-ADDRESS> --haneul-coin-object-id <COIN-OBJECT-ID> --gas-budget <GAS-AMOUNT> --serialize-unsigned-transaction
```

The console responds with the resulting `<TX_BYTES>` value.


:::tip

All other CLI commands that craft a transaction (such as `haneul client publish` and `haneul client call`) also accept the `--serialize-unsigned-transaction` flag and you can use it in the same way.

:::

## Sign the serialized data {#sign}

You can sign the data using the device and programming language of your choice. Haneul accepts signatures for pure Ed25519, ECDSA Secp256k1, ECDSA Secp256r1, and native multisig. To learn more about the requirements of the signatures, see [Haneul Signatures](/concepts/transactions/transaction-auth.mdx).

This example uses the `haneul keytool` command to sign, using the Ed25519 key corresponding to the provided address stored in `haneul.keystore`. This command outputs the signature, the public key, and the flag encoded in Base64. This command is backed by [fastcrypto](https://github.com/GeunhwaJeong/fastcrypto).

`haneul keytool sign --address <HANEUL-ADDRESS> --data <TX_BYTES>`

You receive the following response:

```sh
Signer address: <HANEUL-ADDRESS>
Raw tx_bytes to execute: <TX_BYTES>
Intent: Intent { scope: TransactionData, version: V0, app_id: Haneul }
Raw intent message: <INTENT-MESSAGE>
Digest to sign: <DIGEST>
Serialized signature (`flag || sig || pk` in Base64): <SERIALIZED-SIGNATURE>
```

To ensure the signature produced offline meets Haneul validity rules for testing purposes, you can import the mnemonics to `haneul.keystore` using `haneul keytool import`. You can then sign with it using `haneul keytool sign` and compare the signature results. Additionally, you can find test vectors in `~/haneul/sdk/typescript/test/e2e/raw-signer.test.ts`.

To verify a signature against the cryptography library backing Haneul when debugging, see [sigs-cli](https://github.com/GeunhwaJeong/fastcrypto/blob/4cf71bd8b3a373495beeb77ce81c27827516c218/fastcrypto-cli/src/sigs_cli.rs).

## Execute the signed transaction {#execute}

After you obtain the serialized signature, you can submit it using the execution transaction command. This command takes `--tx-bytes` as the raw transaction bytes to execute (see output of the previous `haneul client transfer` command) and the serialized signature (Base64 encoded `flag || sig || pk`, see output of `haneul keytool sign`). This executes the signed transaction and returns the certificate and transaction effects if successful.

```sh
$ haneul client execute-signed-tx --tx-bytes <TX_BYTES> --signatures <SERIALIZED-SIGNATURE>
```

You receive the following response:

```sh
----- Certificate ----
Transaction Hash: <TRANSACTION-ID>
Transaction Signature: <SIGNATURE>
Signed Authorities Bitmap: RoaringBitmap<[0, 1, 3]>
Transaction Kind : Transfer HANEUL
Recipient : <HANEUL-ADDRESS>
Amount: Full Balance

----- Transaction Effects ----
Status : Success
Mutated Objects:
 - ID: <OBJECT_ID> , Owner: Account Address ( <HANEUL-ADDRESS> )
```

## Alternative: Sign with Haneul Keystore and execute transaction

Alternatively, you can use the active key in Haneul Keystore to sign and output a Base64-encoded sender signed data with flag `--serialize-signed-transaction`. 

```sh
$ haneul client transfer-haneul --to <HANEUL-ADDRESS> --haneul-coin-object-id <COIN-OBJECT-ID> --gas-budget <GAS-AMOUNT> --serialize-signed-transaction
```

The console responds with the resulting `<SIGNED-TX-BYTES>` value.

After you obtain the signed transaction bytes, you can submit it using the `execute-combined-signed-tx` command. This command takes `--signed-tx-bytes` as the signed transaction bytes to execute (see output of the previous `haneul client transfer-haneul` command). This executes the signed transaction and returns the certificate and transaction effects if successful.

```sh
$ haneul client execute-combined-signed-tx --signed-tx-bytes <SIGNED-TX-BYTES>
```
