zkLogin wallets derive a Haneul address from an OAuth credential rather than a traditional private key or recovery passphrase. The user signs in with a provider they already use (Google, Apple, Twitch, and others), and the wallet generates a Haneul address tied to that credential. No seed phrase is created, and no persistent private key is stored by the OAuth provider. [zkLogin is a primitive native to Haneul](/guides/developer/cryptography/zklogin-integration/zklogin), designed to remove the key management burden for users who are new to onchain applications.

## How zkLogin wallets work

At a high level, a zkLogin wallet works as follows:

1. The app generates an [ephemeral key pair](/guides/developer/cryptography/zklogin-integration/zklogin#how-zklogin-works), valid for a limited number of epochs.
2. The user authenticates with an OAuth provider. The provider returns a [JSON Web Token (JWT)](/guides/developer/cryptography/zklogin-integration/zklogin#json-web-token-jwt) that contains a nonce derived from the ephemeral public key.
3. The app or a proving service uses the JWT to generate a zero-knowledge proof (ZKP). The proof confirms the user holds a valid OAuth credential without revealing the credential onchain.
4. The app uses the JWT, a [per-user salt](/guides/developer/cryptography/zklogin-integration/zklogin#user-salt), and the issuer URL to [derive a stable Haneul address](/guides/developer/cryptography/zklogin-integration/zklogin#address-definition) for the user. The same credential always produces the same address for a given app and salt.
5. Transactions are signed with the [ephemeral private key](/guides/developer/cryptography/zklogin-integration/zklogin#ephemeral-private-key) and submitted alongside the ZKP. Validators verify the proof and execute the transaction.

Because zkLogin is a two-factor scheme, an attacker who compromises an OAuth account cannot sign transactions unless they also compromise the per-user salt.

## Enoki

[Enoki](https://enoki.haneul-labs.com/) is a Haneul Labs platform that wraps zkLogin and sponsored transactions behind a simple API. Rather than managing proof generation, salt storage, and OAuth configuration yourself, you register your app on the Enoki Developer Portal, configure your OAuth providers, and use the [`@haneullabs/enoki` SDK](https://docs.enoki.haneul-labs.com/) to handle the rest.

Enoki implements the [Wallet Standard](/standards/wallet-standard) and integrate with [Haneul dApp Kit](https://sdk.haneul-labs.com/dapp-kit) through `registerEnokiWallets`. Once registered, they appear in the standard connection UI alongside any other installed wallets.

## Playtron wallet

The [Playtron](https://www.playtron.one/) wallet is the default zkLogin wallet on the [HaneulPlay0X1](/guides/haneulplay0x1/wallet-integration). Every HaneulPlay0X1 user has a Playtron account, and every Playtron account has an associated zkLogin wallet derived from those credentials.

Games running on the HaneulPlay0X1 must support the Playtron wallet as the default option. Off-device versions of those games should use Haneul dApp Kit to allow users to connect their Playtron wallet through a web interface.

## zkLogin SDK

The [`@haneullabs/haneul/zklogin` module in the Haneul TypeScript SDK](https://sdk.haneul-labs.com/haneul/zklogin) provides utilities for building zkLogin wallets and apps directly, without using a managed service like Enoki. Use this SDK when you need full control over proof generation, salt management, and address derivation.

Install the Haneul TypeScript SDK:

```bash
npm i @haneullabs/haneul
```

### Proof generation

The Haneul TypeScript SDK handles address derivation and signature serialization, but it does not generate ZKPs. Proof generation requires a prover service:

- **Haneul Labs prover:** A publicly accessible proving service maintained by Haneul Labs. Suitable for Testnet and Devnet development. See [zkLogin integration guide](/guides/developer/cryptography/zklogin-integration) for the endpoint and request format.
- **Self-hosted prover:** You run your own prover for production environments where you need full control over the proving infrastructure.
