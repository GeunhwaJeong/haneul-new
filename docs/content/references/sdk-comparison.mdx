This page provides a high-level comparison of the SDKs available for building on Haneul. Use it to find the kit that best matches your language, platform, and use case. For links to each SDK and a fuller list of community projects, see [Haneul and Community SDKs](./haneul-sdks.mdx).

:::info Contribute to this page

This comparison currently lists the SDKs maintained by Haneul Labs. If you maintain a Haneul SDK and want to add it, contributions are welcome. See [Add your SDK](#add-your-sdk) for how to submit a row.

:::

## Haneul SDKs

The following SDKs are developed and maintained by Haneul Labs.

| SDK | Language / platform | Maintainer | Primary use case | Transport | Status |
| --- | --- | --- | --- | --- | --- |
| [TypeScript SDK](https://sdk.haneul-labs.com/typescript) | TypeScript / JavaScript | Haneul Labs | General-purpose client for building apps, scripts, and services | gRPC (recommended), GraphQL, JSON-RPC (deprecated, do not use) | Stable |
| [dApp Kit](https://sdk.haneul-labs.com/dapp-kit) | TypeScript / React | Haneul Labs | Web frontends: wallet connection, hooks, and UI components | Builds on TypeScript SDK | Stable |
| [Rust SDK](https://github.com/GeunhwaJeong/haneul-rust-sdk) | Rust | Haneul Labs | Backend services and high-performance clients | gRPC | Active development |
| [zkSend SDK](https://sdk.haneul-labs.com/zksend) | TypeScript | Haneul Labs | Creating and claiming zkSend links and Stashed functionality | Builds on TypeScript SDK | Stable |
| [DeepBookV3 SDK](../onchain-finance/deepbookv3-sdk/deepbookv3-sdk.mdx) | TypeScript | Haneul Labs | Interacting with the DeepBook onchain order book | Builds on TypeScript SDK | Stable |

## Choosing an SDK

- **Building a web app or wallet integration?** Start with the [dApp Kit](https://sdk.haneul-labs.com/dapp-kit), which builds on the TypeScript SDK and adds React hooks and components for wallet connection.
- **Writing scripts, services, or a Node.js backend in TypeScript?** Use the [TypeScript SDK](https://sdk.haneul-labs.com/typescript) directly. Prefer its gRPC client (`HaneulGrpcClient`) for new projects; the JSON-RPC client is deprecated.
- **Building a performance-sensitive backend in Rust?** Use the [Rust SDK](https://github.com/GeunhwaJeong/haneul-rust-sdk) (gRPC).
- **Working with DeepBook or zkSend?** Use the purpose-built [DeepBookV3 SDK](../onchain-finance/deepbookv3-sdk/deepbookv3-sdk.mdx) or [zkSend SDK](https://sdk.haneul-labs.com/zksend), which layer on top of the TypeScript SDK.
- **Need another language (Python, Go, Kotlin, Swift, Dart, Vue)?** See the community SDKs on the [Haneul and Community SDKs](./haneul-sdks.mdx#community-sdks) page.

## Add your SDK

This comparison is intended to grow with the ecosystem. If you maintain a Haneul SDK, you can add it to a future "Community SDKs" comparison table by opening a pull request against the [Haneul repository](https://github.com/GeunhwaJeong/haneul).

When you submit a row, please include the following so the comparison stays consistent and useful:

| Column | What to provide |
| --- | --- |
| SDK | Name of the SDK, linked to its documentation or repository. |
| Language / platform | The primary language(s) and platform(s) the SDK targets. |
| Maintainer | The individual or organization that maintains the SDK. |
| Primary use case | A short phrase describing what the SDK is best suited for. |
| Transport | The API the SDK uses to communicate with Haneul (for example, GraphQL, JSON-RPC, gRPC), or the SDK it builds on. |
| Status | The maturity of the SDK (for example, Stable, Active development, Maintenance, Experimental). |

For guidance on contributing to Haneul documentation, see [Contribute to Haneul](./contribute/contribution-process.mdx) and the [Style Guide](./contribute/style-guide.mdx).

:::info

Maintenance and community support vary. Research a project's history and support level before committing to using its utilities.

:::
