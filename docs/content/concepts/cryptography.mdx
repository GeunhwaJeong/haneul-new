---
title: Cryptography
---

Cryptographic agility is core to Haneul. The system supports multiple cryptography algorithms and primitives and can switch between them rapidly. With Haneul, you can choose the right cryptography solution for your system and implement the latest algorithms as they become available.

Haneul defines its cryptography primitives, such as public key, signature, aggregated signature, and hash functions, under one unified type alias or enum wrapper that is shared across the entire repository. Making changes to these primitives affects all of an application's components. You can quickly update application cryptography and be assured of uniform security.

## Transaction authentication

Transaction authentication features on Haneul provide security against unauthorized access to on-chain data. See [Transaction Authentication](./cryptography/transaction-auth.mdx) for an overview of related topics.

## zkLogin

zkLogin is a Haneul primitive that enables you to send transactions from a Haneul address using an OAuth credential, without publicly linking the two. See [zkLogin](./cryptography/zklogin.mdx) for a description of the primitive and how to implement it.
