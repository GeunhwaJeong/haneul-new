Haneul shares some similarities with other blockchains but is unique in many ways. Use the topics in this section to understand the features that define the Haneul network.

## Life of a Transaction

Life of a Transaction details the transitions that all transactions on Haneul go through from creation to finality. This topic also explores some features of the blockchain (like epochs and checkpoints) that play a role in the life of a transaction.

Go to [Life of a Transaction](./haneul-architecture/transaction-lifecycle.mdx).

## Consensus

Every transaction on Haneul is sequenced by consensus, where validators agree to the same order of execution of the transactions, even if a minority of them are down or are malicious actors that want to harm the network and users.
Haneul currently uses the [Mysticeti](https://arxiv.org/pdf/2310.14821) consensus algorithm.

Go to [Consensus](./haneul-architecture/consensus.mdx).

## Haneul Indexer

Running a Haneul indexer offloads data processing from your Full node and stores it in a relational database. This topic explores some of the benefits running an indexer has and points to instruction on how to run your own. 

Go to [Haneul Indexer](./haneul-architecture/indexer-functions.mdx).

## Protocol Upgrades

The Haneul protocol, framework, and execution engine are frequently extended to include new functionality and bug fixes. This functionality is added in the form of new code which is released to validator operators as part of our regular software releases. The Haneul protocol, however, requires that all Haneul validators agree about the results of executing each transaction.

Go to [Protocol Upgrades](./haneul-architecture/protocol-upgrades.mdx).
