---
title: Connect and Communicate with the Haneul Network
---

Now that you have [installed Haneul](install.md), [connected to Devnet](../explore/devnet.md), and started to [create smart contracts in Move](move/index.md) and [program Haneul objects](programming-with-objects/index.md), it's time to let your apps talk to Haneul.

The pages in this section provide various options for communicating with Haneul:

* Set up your [Haneul RPC server and use the Haneul JSON-RPC API](json-rpc.md) to interact with a local Haneul network.
* Adhere to the [restrictions placed on JSON types](haneul-json.md) to make them HaneulJSON compatible.
* Interact with the Haneul network via the [Haneul Rust SDK](rust-sdk.md), a collection of Rust language JSON-RPC wrapper and crypto utilities.
* Sign transactions and interact with the Haneul network using the [Haneul TypeScript SDK](https://github.com/GeunhwaJeong/haneul/tree/main/sdk/typescript) built on the Haneul JSON RPC API.
* Run a [Haneul Fullnode](fullnode.md) yourself to store the full Haneul blockchain state and history and qualify as a [potential validator](https://haneul.io/resources-haneul/validator-registration-open/).
* Filter and subscribe to a [real-time event stream](pubsub.md) on your Haneul Fullnode using JSON-RPC notifications via the WebSocket API.
* Optionally, use the [Haneul CLI client](cli-client.md) to start and set up a local Haneul network for contributing changes to the blockchain itself.
